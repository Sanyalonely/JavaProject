2. День 2:
   
* Розроблений клас, що серіалізується, для зберігання параметрів і результатів обчислень;

* Розроблений клас для демонстрації в діалоговому режимі збереження та відновлення стану об'єкта, використовуючи серіалізацію;

* Розроблений клас для тестування коректності результатів обчислень та серіалізації/десеріалізації;

* Написано докладні коментарі для автоматичної генерації документації засобами javadoc.

* Виконато індивідуальне завдання під варіантом 13: Знайти двійкове уявлення цілої частини значення кінетичної енергії
фізичного тіла при заданих значеннях маси та швидкості.

3. День 3:
   
* Забезпечено розміщення результатів обчислень у колекції з можливістю збереження та відновлення даних.

* Розроблено ієрархію класів із використанням шаблону проектування Factory Method (Virtual Constructor), що передбачає подальше розширення за рахунок додавання нових відображуваних класів.

* Розширено ієрархію інтерфейсом "фабрикованих" об'єктів (View), що представляє набір методів для відображення результатів обчислень.

* Реалізовано методи виведення результатів обчислень у текстовому вигляді.

* Розроблено та реалізовано інтерфейс для "фабрикуючого" методу (Viewable), що забезпечує створення об'єктів відображення.

4. День 4

* Розширено ієрархію похідними класами (ViewTable), які реалізують методи для представлення результатів у вигляді текстової таблиці. Забезпечено можливість визначення параметрів відображення таблиці користувачем.

* Продемонстровано заміщення (перевизначення, overriding), поєднання (перевантаження, overloading), динамічне призначення методів (Пізнє зв'язування, поліморфізм, dynamic method dispatch).

* Забезпечено діалоговий інтерфейс із користувачем.

* Розроблено клас для тестування основної функціональності.
  
* Використано коментарі для автоматичної генерації документації засобами javadoc.
